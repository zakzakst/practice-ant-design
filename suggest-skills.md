# 開発環境に便利なエージェントスキル提案

## 📋 テスト・品質保証関連

### 1. テストカバレッジ分析スキル
- 既存のテストコード (`__tests__/`) を分析
- テストが不足している部分を自動検出
- 未テスト関数やコンポーネントの一覧を提示

### 2. テスト生成スキル
- 既存のコンポーネント/関数から Jest テストを自動生成
- `__tests__/` ディレクトリのパターンに合わせてテストコードを作成
- アサーション内容を自動提案

### 3. E2E テスト生成
- React Hook Form や form 関連の統合テストを生成
- ユーザーインタラクションをシミュレートするテストを自動作成

---

## 🔄 API・データフロー関連

### 4. OpenAPI 連携スキル
- `openapi.yaml` と `orval` 設定を解析
- 型安全なフェッチャーを自動生成
- API 定義の変更を検出して、既存コードの更新を提案

### 5. API 統合テスト生成
- MSW + SWR を使った API 呼び出しのテストを作成
- モック設定の自動化
- `mocks/` ディレクトリを活用したハンドラー生成

---

## ⚙️ フォーム・状態管理関連

### 6. バリデーション統合スキル
- React Hook Form + Zod/Yup のセットアップを自動化
- スキーマの競合検出・統合
- 複数の react-hook-form 実装パターンの分析

### 7. Reducer パターン生成
- 複雑な状態管理用の `useReducer` テンプレート自動生成
- `models/reducer.ts` のパターンを参考にした実装提案
- Action / State type の自動定義

---

## 🎨 複雑な UI ライブラリ対応

### 8. 複雑ライブラリ初期化スキル
- react-flow、dnd-kit、Konva などの基本セットアップを自動化
- 既存実装 (`src/app/` 配下) からベストプラクティスを抽出
- サンプルコード・テンプレートの生成

### 9. shadcn/ui カスタマイズスキル
- 既存コンポーネント (`src/components/ui/`) を分析
- 必要なカスタマイズを提案
- 新規コンポーネント追加時のテンプレート生成

---

## 📁 コード整理・最適化

### 10. コード重複検出スキル
- 複数の react-hook-form 実装パターン (react-hook-form ~ react-hook-form4) から共通部分を抽出
- 重複コードの統合・ライブラリ化を提案

### 11. パス・import 最適化
- 深くなったパス構造を整理 (`src/app/` 配下に多数のフォルダが存在)
- import 文を最適化
- モジュール構成の改善提案

---

## 🚀 プロジェクト固有

### 12. 練習用サンプル生成スキル
- `__tests__/` のパターンに合わせて新しい技術スタックの練習コードを自動生成
- 新しいライブラリ追加時に、既存パターンに準拠した学習用サンプルを作成
- READMEや解説コメントも自動生成

---

## 総括

このプロジェクトは **学習・練習用** に見えるため、以下の優先度を推奨：

**高優先度:**
1. 練習用サンプル生成スキル
2. テスト生成スキル
3. OpenAPI 連携スキル

**中優先度:**
4. 複雑ライブラリ初期化スキル
5. バリデーション統合スキル
6. コード重複検出スキル

**低優先度:**
7. その他の補助スキル
